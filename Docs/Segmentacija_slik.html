Segmentacija slik

Cilj segmentacije

  • sliko razdeliti v smiselne regije
    • ločitev predmeta zanimanja od ozadja
    • sliko razdeliti na semantično smiselne regije
    • piksle slike združiti v gruče za lažjo obdelavo (superpiksel segmentacija)

Sorodni problem

Iskanje robov regij je načeloma enakovredno iskanju regij samih. Zato lahko takšne algoritme pogosto obravnavamo v isti kategoriji - segmentacija.

Pristopi

Pragovna segmentacija

Pragovna segmentacija je preprosta oblika binarne segmentacije - sliko razdelimo v dve regiji.

  1. izberemo prag $v_t$
  2. če $I[r,c] < v_t$ potem segment 1
  3. če $I[r,c] \ge v_t$ potem segment 2

Prednosti:

  • hitra in preprosta metoda Slabosti:
  • izbrati moramo primeren prag
  • sliko lahko segmentiramo le na dva dela
  • osnovni podatki pogosto niso deljivi v dva razreda s pragom

Kako izberemo prag?

Metoda Otsu

https://en.wikipedia.org/wiki/Otsu's_method

Izbran prag naj minimizira varianco vrednosti v obeh segmentih. Takšen prag poiščemo z izčrpnim iskanjem:

  1. uredimo vse vrednosti po vrsti
  2. vsako izmed unikatnih vrednosti preizkusimo kot prag
  3. kot prag izberemo vrednost z najmanjšo varianco

Metodo lahko aproksimiramo z uporabo histograma.

Povezave sortiramo. Nato za vsako povezavo od najmanjše do največje pogledamo, ali bomo združili segmente, v katerih sta njene vozlišča.

Imejmo povezavo med vozliščema $v_i$ in $v_j$, utež povezave pa je $w(v_i,v_j)$. Če vozlišči nista v istem segmentu, potem segmenta združimo kadar j: $$ w(v_i,v_j) \le max(int(S_i), int(S_j)) $$

$S_i$ in $S_j$ sta grafa - segmenta, ki ju želimo združiti. $Int$ pa je maksimalna utež povezav v minimalnem vpetem drevesu teh grafov (pozor, to ni maksimalna utež povezav v tem grafu).

In [51]:
# tukaj poiscemo maksimalno povezavo v vpetem drevesu
# v bistvu malce goljufamo - ker imamo povezave urejene po utezem
# poiscemo utez zadnje povezave, ki jo moramo dodati
# da povezemo vsa vozlisca
def int_fun(povezave):
    vozlisca = np.unique(povezave[:,:2].ravel())
    for n in range(0,povezave.shape[0]):
        m = vozlisca==povezave[n,0]
        m += vozlisca==povezave[n,1]
        vozlisca[m] = -1
        if vozlisca.max() == -1:
            Int_c_i = povezave[n,2]
            return Int_c_i

Ker je ta vrednost nestabilna kadar je število vozlišč majhno, dodamo še konstanto. $$ w(v_i,v_j) \le max(int(S_i)+ {k \over |S_i| }, int(S_j)+ {k \over |S_j| }) $$

Spremenljivka $k$ je parameter metode s katerim nadzorujemo število tvorjenih segmentov. Delimo jo z številom vozlišč v grafu - segmentu. Kadar je število vozlišč v segmentu veliko, $k$ nima bistvenega vpliva na rezultat primerjave.

for q in range(0, povezave.shape[0]):
    v_i, v_j, w = povezave[q]
    s_i = segmentacija[v_i]
    s_j = segmentacija[v_j]
    # vozlisci sta ze v istem segmentu
    if s_i == s_j:
        continue
    
    c_i = segmentacija == s_i
    c_j = segmentacija == s_j
    
    c_i_seg = (seg_povezav == s_i).min(1)
    c_j_seg = (seg_povezav == s_j).min(1)
    
    Int_c_i = k/c_i.sum()
    if c_j_seg.sum() > 0:
        Int_c_i += int_fun(povezave[c_i_seg,:])
        
    Int_c_j = k/c_j.sum()
    if c_j_seg.sum() > 0:
        Int_c_i += int_fun(povezave[c_j_seg,:])
   
    MInt = np.min([Int_c_i,Int_c_j])

    if w <= MInt:
        seg_povezav[c_j_seg] = s_i
        segmentacija[c_j] = s_i

Indeksi segmentov so trenutno pomešani, kako so bili pač združevani v metodi. Tu jih še enkrat indeksiramo, lepo po vrsti od 0 naprej.

KMeans lahko izboljšamo, če namesto samo barvnih vrednosti dodamo tudi lokacije pikslov. Tako so dobljeni segmenti bližje skupaj tudi v prostoru slike (ne samo barvnem prostoru).

Prvi parameter algoritma je obtežitev prostorskih koordinat v primerjavi z barvami. Odvisno on naših želja optimizacije bomo želeli večjo pomembnost dati prostorskim koordinatam ali pa barvnim vrednostim.
